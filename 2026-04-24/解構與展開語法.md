### 一、解構賦值（Destructuring）

**定義**：從陣列或物件中，依照「結構形狀」取出值並綁定到變數，是一種語法糖，底層仍是逐一讀取屬性。

---

#### 1. 物件解構

```ts
interface User {
  id: number
  name: string
  email: string
  isActive: boolean
}

const user: User = {
  id: 1,
  name: 'Alice',
  email: 'alice@example.com',
  isActive: true
}

// 基本解構
const { name, email } = user
// 等價於：
// const name = user.name
// const email = user.email
```

**重命名**：解構時可以改名（避免變數衝突、提升可讀性）

```ts
const { name: userName, email: userEmail } = user
// userName === 'Alice'
// name / email 不存在於當前作用域
```

**預設值**：屬性為 `undefined` 時使用預設值（注意：`null` 不會觸發預設值）

```ts
const { nickname = 'Anonymous' } = user
// user 沒有 nickname 屬性 → nickname === 'Anonymous'

const { name = 'Guest' } = { name: null }
// name === null（不是 'Guest'，因為 null !== undefined）
```

**重命名 + 預設值 同時使用**

```ts
const { role: userRole = 'viewer' } = user
// user 沒有 role → userRole === 'viewer'
```

---

#### 2. 陣列解構

```ts
const colors: string[] = ['red', 'green', 'blue']

const [primary, secondary, tertiary] = colors
// primary === 'red'
// secondary === 'green'
// tertiary === 'blue'

// 跳過元素
const [, , third] = colors
// third === 'blue'

// 預設值
const [a = 0, b = 0, c = 0, d = 0] = [1, 2]
// a=1, b=2, c=0, d=0

// 交換變數（經典應用）
let x = 1
let y = 2
;[x, y] = [y, x]
// x === 2, y === 1
```

---

#### 3. 巢狀解構

```ts
interface ApiResponse {
  data: {
    user: {
      id: number
      profile: { age: number; city: string }
    }
    token: string
  }
  meta: { timestamp: number }
}

const response: ApiResponse = {
  data: {
    user: {
      id: 1,
      profile: { age: 25, city: 'Taipei' }
    },
    token: 'abc123'
  },
  meta: { timestamp: 1713888000 }
}

const {
  data: {
    user: {
      id,
      profile: { city }
    },
    token
  }
} = response

// id === 1, city === 'Taipei', token === 'abc123'
// 注意：data、user、profile 這些中間變數不存在
```

**常見誤解**：`data: { user }` 這裡的 `data:` **不是重命名成 data**，而是「進入 data 屬性內部解構」。真正要建立 `data` 變數必須寫：

```ts
const { data, data: { user } } = response
// 兩個都要
```

---

#### 4. 函式參數解構

```ts
interface FetchOptions {
  url: string
  method?: 'GET' | 'POST'
  headers?: Record<string, string>
  timeout?: number
}

// 直接在參數位置解構 + 預設值
function fetchData({
  url,
  method = 'GET',
  headers = {},
  timeout = 5000
}: FetchOptions) {
  console.log(`${method} ${url} (timeout: ${timeout}ms)`)
}

fetchData({ url: '/api/users' })
// GET /api/users (timeout: 5000ms)
```

**好處**：不用 `options.url` 這種寫法、參數順序不重要、自然支援 optional 參數。

### 二、展開運算子（Spread `...`）

**定義**：將可迭代物件（陣列、物件）「展開」成獨立元素／屬性。

---

#### 1. 陣列展開

```ts
const nums1 = [1, 2, 3]
const nums2 = [4, 5, 6]

// 合併陣列（產生新陣列，不改原陣列）
const merged = [...nums1, ...nums2]
// [1, 2, 3, 4, 5, 6]

// 複製陣列（淺拷貝）
const copy = [...nums1]

// 當作函式參數
Math.max(...nums1) // 等同於 Math.max(1, 2, 3)

// 字串也是可迭代
const chars = [...'abc'] // ['a', 'b', 'c']
```

---

#### 2. 物件展開

```ts
const base = { name: 'Alice', age: 25 }
const extra = { age: 26, city: 'Taipei' }

// 合併物件（後者覆蓋前者）
const merged = { ...base, ...extra }
// { name: 'Alice', age: 26, city: 'Taipei' }

// 複製並修改（Redux/Pinia 常見模式）
const updated = { ...base, age: 30 }
// 原 base 不變，updated 為 { name: 'Alice', age: 30 }
```

**重要：淺拷貝（Shallow Copy）**

```ts
const original = {
  name: 'Alice',
  address: { city: 'Taipei' }
}

const copy = { ...original }
copy.address.city = 'Tokyo'
console.log(original.address.city) // 'Tokyo' ← 被改了！

// 因為 address 仍是同一個參考
// 深拷貝請用 structuredClone() 或 JSON 方案
```

---

### 三、其餘運算子（Rest `...`）

**定義**：「收集剩下的」——語法長得跟展開一樣，但語意相反（展開是打散、Rest 是蒐集）。

---

#### 1. 陣列 Rest

```ts
const [first, ...rest] = [1, 2, 3, 4, 5]
// first === 1
// rest === [2, 3, 4, 5]
```

#### 2. 物件 Rest

```ts
const { id, ...profileWithoutId } = {
  id: 1,
  name: 'Alice',
  age: 25,
  email: 'alice@example.com'
}
// id === 1
// profileWithoutId === { name, age, email }
```

**應用**：移除某個敏感欄位後轉發資料

```ts
function sanitizeUser(user: User & { password: string }) {
  const { password, ...safe } = user
  return safe // 不含 password
}
```

#### 3. 函式參數 Rest

```ts
// 不定參數函式
function sum(...nums: number[]): number {
  return nums.reduce((acc, n) => acc + n, 0)
}

sum(1, 2, 3)    // 6
sum(1, 2, 3, 4) // 10
```

---

### 四、展開 vs Rest 差異一表搞懂

| 特徵    | 展開 Spread               | 其餘 Rest                    |
| ----- | ----------------------- | -------------------------- |
| 位置    | 通常在**右側**（值的位置）         | 通常在**左側**（宣告的位置）           |
| 語意    | 打散 / 鋪平                 | 蒐集 / 打包                    |
| 範例    | `[...arr1, ...arr2]`    | `const [a, ...rest] = arr` |
| 物件應用  | 合併：`{...a, ...b}`       | 剔除：`const {x, ...y} = obj` |
| 函式內位置 | 呼叫時：`fn(...args)`       | 宣告時：`function fn(...args)` |

**語法長得一樣，但看位置就知道是哪個**。

### 五、與昨日 Vue 3 響應式的連結（回頭解謎）

昨天說：

> 解構 `reactive` 物件後會失去響應性。

**為什麼？** 現在用今天學的知識重新解釋：

```ts
const state = reactive({ count: 0, name: 'Alice' })

// 解構相當於：
const count = state.count // 把「目前的值 0」複製到新變數
const name = state.name   // 把「目前的值 'Alice'」複製到新變數
```

- `state.count` 的 get 操作確實觸發了 Proxy 的依賴收集
- 但 **得到的是值 `0`**（基本型別，值傳遞）
- `count` 變數只是記住「0 這個數字」，跟 `state` 的 Proxy 再也沒有關係
- 之後 `state.count = 5` 時，`count` 不會知道

**`toRefs` 的解法**：

```ts
import { toRefs } from 'vue'

const { count, name } = toRefs(state)
// count 現在是 Ref<number>，本身是物件
// 修改 count.value 會反映回 state.count，反之亦然
```

`toRefs` 把 `reactive` 物件的每個屬性包成 `ref`，於是解構後每個變數都是**物件參考**，保持與原資料的連結。

### 六、TypeScript 整合重點

#### 1. 解構 + 型別標注

```ts
interface Product {
  id: number
  name: string
  price: number
}

// 正確：在解構外標注
const { name, price }: Product = { id: 1, name: 'Pen', price: 20 }

// 錯誤寫法：這是重命名，不是型別
// const { name: string, price: number } = ...  ← 編譯錯誤！
```

#### 2. Rest 物件的型別推導

```ts
const user: Product = { id: 1, name: 'Pen', price: 20 }
const { id, ...rest } = user
// rest 的型別被 TS 自動推導為 { name: string; price: number }
```

#### 3. 函式 Rest 參數的型別

```ts
function merge<T>(first: T, ...rest: T[]): T[] {
  return [first, ...rest]
}

const result = merge(1, 2, 3) // number[]
```

### 七、小結：什麼時候用哪個

| 情境          | 用什麼                                         |
| ----------- | ------------------------------------------- |
| 取出物件/陣列部分屬性 | 解構                                          |
| 合併多個陣列/物件   | Spread                                      |
| 不可變更新（複製再改） | Spread                                      |
| 移除特定屬性      | Rest                                        |
| 不定個數函式參數    | Rest                                        |
| 解構 reactive | ⚠ 搭配 `toRefs`                               |
| 傳遞 Props    | `<Child v-bind="propsObject" />` 或展開 script |

## 作業說明

### 作業主題：實作「使用者資料處理工具函式庫」

使用 **ES6+ 解構、Spread、Rest**（加上 TypeScript 型別定義），實作一組處理使用者資料的純工具函式。

---

### 情境背景

ihouseBMS 後端回傳如下使用者資料：

```ts
interface RawUser {
  id: number
  name: string
  email: string
  password: string        // 敏感欄位，不可外流
  role: 'admin' | 'user'
  profile: {
    age: number
    city: string
    avatar?: string
  }
  permissions: string[]
}
```

---

### 功能需求

#### 1. `sanitizeUser(user)` — 移除敏感欄位

要求：
- 使用**物件 Rest** 移除 `password`
- 回傳的型別不能再包含 `password`
- 原物件不可被修改（純函式）

```ts
const safe = sanitizeUser(rawUser)
// safe 型別：Omit<RawUser, 'password'>
// safe.password ← 型別錯誤
```

---

#### 2. `extractProfile(user)` — 取出常用欄位

要求：
- 使用**巢狀解構**一次取出 `name`、`profile.age`、`profile.city`
- 將取出的欄位重命名為 `userName`、`userAge`、`userCity`
- 回傳一個新的扁平物件

```ts
extractProfile(rawUser)
// { userName: 'Alice', userAge: 25, userCity: 'Taipei' }
```

---

#### 3. `mergeUsers(base, patch)` — 合併使用者資料

要求：
- 使用**物件 Spread** 進行淺合併
- `patch` 的欄位會覆蓋 `base`
- 若 `patch.profile` 存在，也要合併 profile（示範**巢狀合併**的正確寫法）
- 原物件皆不可被修改

```ts
mergeUsers(
  { ...base, profile: { age: 25, city: 'Taipei' } },
  { name: 'Bob', profile: { city: 'Tokyo' } }
)
// 預期：name 變 Bob、profile.age 仍為 25、profile.city 變 Tokyo
```

---

#### 4. `combinePermissions(...groups)` — 合併多組權限

要求：
- 使用**函式 Rest** 接收任意多個權限字串陣列
- 使用**陣列 Spread** 合併並**去重**
- 以字母序排列後回傳

```ts
combinePermissions(['read', 'write'], ['write', 'delete'], ['read'])
// ['delete', 'read', 'write']
```

---

#### 5. `cloneAndUpdate(user, updates)` — 不可變更新

要求：
- 使用 Spread 複製 `user` 並套用 `updates`
- 原 `user` 不可被修改
- 驗證：呼叫後 `user` 深度比對仍與原本相同

```ts
const newUser = cloneAndUpdate(rawUser, { name: 'Alice Wang' })
// rawUser 不變、newUser.name === 'Alice Wang'
```
