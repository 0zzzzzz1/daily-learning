## 與 ihouseBMS 專案的對照分析

> 根據實際專案開發中遇到的問題，標記出學習計畫的優先補強項目。

| 在專案中遇到的問題 | 根因技術缺口 | 對應學習月份 |
|--------------------|-------------|-------------|
| 修改 reactive 物件後畫面沒更新 | Vue 3 響應式陷阱（解構失去響應性）| M1 |
| 不知道何時用 computed 還是 method | computed 快取機制不理解 | M1 |
| 元件 Props 型別用 `any`，維護困難 | TypeScript 與 Vue 整合 | M3 |
| 重複邏輯分散在多個元件 | Composable 設計能力不足 | M2 |
| 樣式命名衝突，CSS 難維護 | Tailwind 未系統化使用 | M4 |
| API 回傳格式沒有型別保護 | TS 泛型與 API 型別定義 | M3 |
| 改了 A 功能壞了 B 功能，難以察覺 | 缺乏 Unit Test | M5 |
| 頁面切換後資料遺失 | Vue Router + 狀態管理整合 | M2 / M6 |
