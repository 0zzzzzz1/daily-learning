| Lifecycle Hooks 完整時間軸 | 3.8 → | 鞏固，應用到 W3 作業 |

### W3 作業預告

- 使用 `ref` / `reactive` 恰當組織資料結構

## 本週自省

**W2 的核心轉變**：從「會用 API」到「理解時間軸上的位置」。

W1 結束時，我「認識」了 `ref`、`reactive`、`computed`、`watch`——能在範例中使用它們。W2 的學習讓我開始理解這些 API「在何時執行」、「執行順序為何」、「非同步操作的邊界在哪裡」。

這種轉變用一句話概括：**「能用 API」升級到「知道為什麼這樣設計」**。

下週（W3）的挑戰：能否在不查文件的情況下，**解釋**每個 API 的設計原因，而不只是「套用」它？
