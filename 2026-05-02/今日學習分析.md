## 概念鞏固：今日關鍵判斷

**問：為什麼 `const { length } = reactive([])` 之後 `watch(length, ...)` 不會觸發？**

因為 `reactive([])` 雖然讓整個陣列具有響應性，但 `const { length } = reactive([])` 是「解構」操作，等同於 `const length = reactive([]).length`，此時 `length` 只是一個普通的 `number`（例如 `0`），已經和響應式系統斷開連結。`watch` 接到的是靜態的 `0`，不是響應式的屬性。

正確做法：
```typescript
const items = reactive<string[]>([])
watch(() => items.length, (newLen) => {
  console.log('length changed to', newLen)
})
```
使用 **getter 函式**包裹，Vue 的依賴追蹤會在執行 getter 時讀到 `items.length`，並建立響應式連結。

---

**問：`watch` 的 `flush: 'post'` 有什麼用？什麼時候需要它？**

Vue 的 `watch` 預設在 `flush: 'pre'`，意思是「在 Vue 更新 DOM 之前」執行回呼。若你在回呼中需要**讀取 DOM 的最新狀態**（例如讀取更新後的元素高度），就需要 `flush: 'post'`：

```typescript
watch(
  items,
  () => {
    // 此時 DOM 已經反映 items 的最新狀態
    console.log(listEl.value?.scrollHeight)
  },
  { flush: 'post' }
)
```

等價於 `watchPostEffect`（Vue 3.2+ 提供的語法糖）。
