## 學習前 vs 學習後：Lifecycle Hooks 的認知更新

| 概念 | 學習前的認知 | 學習後的認知 |
|------|------------|------------|
| `onMounted` 的用途 | 「就是 mounted，元件出現的時候執行」| 確認是 DOM 就緒後的第一個安全操作時機；SSR 環境中**不執行** |
| `onUpdated` | 「資料更新後執行，可以在裡面改資料」 | **不能在 `onUpdated` 裡修改響應式資料**，會造成無限更新循環 |
| `watchEffect` 的執行時機 | 「和 `onMounted` 差不多，都是掛載後執行」 | `watchEffect` **早於** `onMounted`（在 setup 階段就執行）|
| `watchEffect` + `await` | 以為整個 callback 都能追蹤依賴 | 依賴追蹤**只在第一個 await 之前的同步程式碼**有效 |
| `onCleanup` | 不清楚有這個 API | 理解為「下次觸發前執行的清理函式」，與 `AbortController` 搭配是防止 Race Condition 的標準模式 |
| Composable 中的 Lifecycle | 「Lifecycle 只能在元件裡用」 | Composable 中也可以呼叫 `onMounted` / `onUnmounted`，Vue 會自動綁定到呼叫它的元件 |

## 概念鞏固：用自己的話解釋

**問：為什麼 Composable 中可以呼叫 `onMounted` / `onUnmounted`？**

因為 Vue 的 Composition API 基於「呼叫上下文」來綁定生命週期鉤子。當你在 `<script setup>` 中呼叫一個 Composable 時，Composable 內部呼叫的 `onMounted` 會被 Vue 自動關聯到當前正在設置的元件（Active Instance）。這是 Vue 3 的設計精髓之一：**邏輯的複用性**，把「何時清理」封裝在 Composable 裡，不需要元件手動管理。

```typescript
// useEventListener.ts
function useEventListener(target: EventTarget, event: string, handler: EventListener) {
  // 這兩行會自動綁定到「呼叫這個 Composable 的元件」
  onMounted(() => target.addEventListener(event, handler))
  onUnmounted(() => target.removeEventListener(event, handler))
}
```

---

**問：`watch + immediate: true` 比 `onMounted + watch` 好在哪裡？**

`onMounted + watch` 有兩個問題：
1. **邏輯重複**：初始載入和後續重新載入幾乎是相同的程式碼，維護時要改兩處
2. **SSR 不友好**：`onMounted` 在伺服器端不執行，初始資料在 SSR 中拿不到

`watch + immediate: true` 的優點：
1. **單一來源**：初始執行和後續響應走同一個 callback，邏輯一致
2. **早於 onMounted 執行**：`immediate` 在 setup 階段觸發，若有 SSR 框架（如 Nuxt）可以配合伺服器端資料獲取
3. **搭配 onCleanup 更簡潔**：只需要一個 cleanup 函式

## 今日反思

今天的學習補完了一個重要的「隱性知識」——**生命週期**。

在過去幾天的學習中，我一直知道 `onMounted`、`watch`、`watchEffect` 這些 API，也能在範例中使用它們。但今天第一次真正把它們放在「時間軸」上理解：

```
setup() → onBeforeMount() → [DOM 建立] → onMounted() → ... → onBeforeUnmount() → onUnmounted()
```

這條時間軸讓很多之前「隱約知道但說不清楚」的問題有了答案：
- 為什麼 `watchEffect` 的第一次執行可能讀不到 DOM？（因為它在 `onMounted` 之前）
- 為什麼 SSR 環境中 `onMounted` 裡的資料會消失？（伺服器端沒有 DOM 生命週期）
- 為什麼 Composable 可以有 `onMounted`？（Vue 追蹤當前的 Active Instance）

**「知道一個 API」和「知道它在時間軸上的位置」是兩件完全不同的事。**

> W2-W3 橋接週若用一句話總結：「非同步 × 響應式 × 生命週期——三者組合，才是真正的 Vue 3。」
