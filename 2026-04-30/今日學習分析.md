### Vue 3 `watch` 的非同步使用

| | 學習前 | 學習後 |
|---|---|---|
| **watch + immediate** | 知道 `immediate` 的存在，但實際上幾乎不用 | 理解 `watch + immediate: true` 可以完全取代 `onMounted`，且能同時響應後續變更 |
| **deep: true 的必要性** | 不清楚什麼時候需要 | 當監聽的是 ref 包裝的「物件」時，內部屬性變更不會觸發 watch，必須加 `deep: true` |
| **watch vs watchEffect** | 不清楚差別 | 今日加深理解：`watch` 更精確（明確指定來源）；`watchEffect` 自動追蹤（用於副作用） |

## 概念鞏固：用自己的話解釋

**問：`watch(filters, callback, { deep: true, immediate: true })` 的三個部分各自的意義？**

- **`filters`**：監聽這個 ref 物件，當它「或它的子屬性」有變化時，執行 callback
- **`deep: true`**：如果沒有這個選項，Vue 只監聽 `filters` 本身（ref 的 `.value` 是否被替換），不會偵測到 `filters.value.type` 這樣的子屬性變化
- **`immediate: true`**：元件初始化後立刻執行一次 callback，不需要等到「值真的改變」才執行 — 等同於同時扮演 `onMounted` 的角色

---

**問：Skeleton UI 的 `shimmer` 動畫為何需要 `background-size: 200%`？**

因為動畫的原理是讓「比元素更寬的背景」從右向左滑動。如果背景寬度等於元素，從 0 到 100% 滑動只能讓漸層移動半個身位，效果不明顯。設成 200% 後，從 `200% 0` 滑到 `-200% 0` 就讓整條漸層完整地掃過元素，產生「光從右往左掃」的視覺效果。

---

**問：為什麼 `PropertyCard` 和 `SkeletonCard` 的外層尺寸結構要一致？**

因為 `PropertyList` 在同一個 grid 格子裡，用 `v-if="isLoading"` 決定顯示哪個。如果兩者尺寸不一樣，載入完成後畫面會「跳動」— 這就是所謂的 CLS（Cumulative Layout Shift）。Skeleton 的設計目標就是讓版面在資料載入前後保持穩定，所以必須維持相同的空間佔位。
