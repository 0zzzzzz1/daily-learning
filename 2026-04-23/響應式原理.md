## 知識說明

### Vue 3 響應式原理（Reactivity System）

Vue 3 的響應式系統建立在 **Proxy** 之上，完全重寫自 Vue 2 的 `Object.defineProperty` 方案，帶來更強大且靈活的響應式能力。

---

#### 核心概念

**1. `ref` vs `reactive`**

| 特性 | `ref` | `reactive` |
|------|-------|-----------|
| 適用型別 | 基本型別（string、number、boolean）或物件 | 物件、陣列 |
| 存取方式 | 需透過 `.value` | 直接存取屬性 |
| TypeScript 型別 | `Ref<T>` | `Reactive<T>` |
| 解構後響應性 | 保留（需 `toRefs`） | 解構後**失去**響應性 |

```ts
import { ref, reactive } from 'vue'

// ref 範例
const count = ref<number>(0)
count.value++ // 必須透過 .value 操作

// reactive 範例
const state = reactive<{ name: string; age: number }>({
  name: 'Alice',
  age: 25
})
state.age++ // 直接操作屬性
```

---

**2. `computed` — 計算屬性**

`computed` 會快取結果，只有依賴的響應式資料改變時才重新計算，適合用於衍生資料。

```ts
import { ref, computed } from 'vue'

const price = ref<number>(100)
const quantity = ref<number>(3)

const total = computed<number>(() => price.value * quantity.value)
// total.value === 300，且自動追蹤 price 與 quantity
```

---

**3. `watch` vs `watchEffect`**

- **`watch`**：明確指定監聽目標，可取得新舊值，惰性執行（預設不立即執行）。
- **`watchEffect`**：自動追蹤函式內用到的所有響應式資料，立即執行一次。

```ts
import { ref, watch, watchEffect } from 'vue'

const keyword = ref<string>('')

// watch：明確監聽，可獲取前後值
watch(keyword, (newVal, oldVal) => {
  console.log(`從 "${oldVal}" 變成 "${newVal}"`)
})

// watchEffect：自動追蹤依賴
watchEffect(() => {
  console.log('keyword 目前是：', keyword.value)
})
```

---

**4. Proxy 底層機制**

Vue 3 使用 `Proxy` 攔截物件的 `get`（依賴收集）與 `set`（觸發更新）操作：

```ts
// 簡化示意（非 Vue 原始碼）
function reactive<T extends object>(target: T): T {
  return new Proxy(target, {
    get(obj, key) {
      track(obj, key) // 收集依賴
      return Reflect.get(obj, key)
    },
    set(obj, key, value) {
      Reflect.set(obj, key, value)
      trigger(obj, key) // 通知更新
      return true
    }
  })
}
```

---

**5. 常見陷阱**

- 解構 `reactive` 物件後失去響應性 → 使用 `toRefs()` 解決
- 直接替換 `reactive` 物件整體 → 響應性中斷，應只修改屬性
- `ref` 在 template 中自動解包（不需 `.value`），但在 JS 中需要

### TypeScript 整合重點

使用 TypeScript 時，善用泛型明確標注型別：

```ts
interface User {
  id: number
  name: string
  isActive: boolean
}

const user = ref<User>({
  id: 1,
  name: 'Alice',
  isActive: true
})
```

---

### Tailwind CSS 連結

響應式資料的狀態變化，通常會透過 **動態 class 綁定** 反映在 UI 樣式上：

```vue
<template>
  <button
    :class="[
      'px-4 py-2 rounded font-medium transition-colors',
      isActive
        ? 'bg-blue-500 text-white hover:bg-blue-600'
        : 'bg-gray-200 text-gray-700 hover:bg-gray-300'
    ]"
    @click="isActive = !isActive"
  >
    {{ isActive ? '啟用中' : '已停用' }}
  </button>
</template>

<script setup lang="ts">
import { ref } from 'vue'
const isActive = ref<boolean>(false)
</script>
```

## 作業說明

### 作業主題：實作一個「任務清單」元件

使用 **Vue 3 Composition API + TypeScript + Tailwind CSS** 實作一個具備響應式功能的任務清單元件。

---

### 功能需求

1. **新增任務**
   - 提供輸入框，輸入後按 Enter 或點擊按鈕新增任務
   - 任務名稱不可為空（需驗證）

2. **切換完成狀態**
   - 點擊任務可切換「未完成 / 已完成」狀態
   - 已完成的任務顯示刪除線樣式

3. **刪除任務**
   - 每筆任務右側有刪除按鈕

4. **統計資訊（`computed` 應用）**
   - 顯示「共 N 筆 / 已完成 M 筆」
   - 使用 `computed` 計算，不可直接用變數紀錄

5. **篩選功能（`watch` 應用）**
   - 提供「全部 / 未完成 / 已完成」三種篩選
   - 使用 `watch` 監聽篩選條件，輸出 log 記錄切換行為

---

### 技術規範

| 項目 | 要求 |
|------|------|
| 響應式資料 | 任務列表使用 `reactive`，輸入框使用 `ref` |
| TypeScript | 定義 `Task` interface，所有變數標注型別 |
| Tailwind CSS | 全程使用 Tailwind 撰寫樣式，禁止 inline style |
| Computed | 至少使用一個 `computed` 計算統計資訊 |
| Watch | 監聽篩選條件，於 console 輸出切換 log |
